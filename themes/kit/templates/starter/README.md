## How It Works

Other templates give you someone else's layout. This one gives you a blank page and an AI that builds exactly what you ask for.

<table>
<tr>
<td width="50%" valign="top">

### 1. Tell AI what you want

Open [Hugo Chat](https://hugo.chat/?utm_source=github&utm_medium=readme&utm_content=how_it_works_starter) and describe your site in plain English:

> *"Build me a landing page for my SaaS with a hero, three feature columns, pricing table, and a contact form"*

> *"Create a research lab site with team profiles, a publications list, and a news feed"*

> *"I need a portfolio with a filterable project grid and an about page"*

Hugo Chat generates structured pages with correct front matter, blocks, and content — ready to paste in.

**Hugo Chat** — come back anytime to generate new pages or refine existing ones.

Push to GitHub. Your site builds and deploys automatically via GitHub Pages. Zero cost, zero config.

## What You Can Build

This is a blank canvas with the full HugoBlox engine behind it. Add any combination of content types:

| Content type | URL | Use it for |
| :--- | :--- | :--- |
| **Landing pages** | any path | Hero sections, feature grids, CTAs, pricing — assembled from blocks |
| **Blog** | `/blog/` | Posts with dates, tags, categories, reading time |
| **Projects** | `/projects/` | Software, research projects, portfolio items |
| **Publications** | `/publications/` | Papers, articles, preprints with citation metadata |
| **Events** | `/events/` | Talks, webinars, workshops, conferences |
| **Datasets** | `/datasets/` | Research data with DOIs and download links |
| **Case studies** | `/case-studies/` | Customer stories, project showcases, impact reports |
| **Changelog** | `/changelog/` | Product updates, release notes, version history |
| **Courses** | `/courses/` | Lessons, tutorials, learning paths with breadcrumbs |
| **Docs** | `/docs/` | Documentation with sidebar navigation |
| **FAQ** | `/faq/` | Frequently asked questions |
| **Slides** | `/slides/` | Presentations from Markdown via reveal.js |

Every content type supports tags, categories, search, RSS, related content, and social sharing out of the box.

## 🚀 Get Started

### Step 1: Deploy Your Site

**Option A: Launch in browser** (fastest — no install needed)

> [!TIP]
> Deploy a live site in 60 seconds — no software to install:
> [**Deploy Starter free**](https://hugoblox.com/templates/starter?utm_source=github&utm_medium=readme&utm_content=get_started)

**Option B: Use the CLI**

```bash
# Requires Hugo Extended & Node.js
npx hugoblox create site --template starter
```

### Step 2: Build Pages With AI

Open [Hugo Chat](https://hugo.chat/?utm_source=github&utm_medium=readme&utm_content=step2_starter) and describe the pages you want. Paste the generated files into your `content/` folder. Push to GitHub — your site updates automatically.

### Step 3: Customize Visually

Install [Ownable CMS](https://marketplace.visualstudio.com/items?itemName=ownable.ownable) in VS Code to edit pages visually — rearrange blocks, update content, and preview changes without touching code.

![Ownable CMS in Action](https://raw.githubusercontent.com/HugoBlox/kit/main/.github/media/studio/slide-1.webp)
*Ownable CMS: Drag-and-drop page builder inside VS Code.*

> [!NOTE]
> **New to Hugo?** No problem. You don't need to know Hugo — build with AI or edit visually. Hugo is the engine under the hood that makes your site fast, secure, and free to host. [Learn more →](https://docs.ownable.dev/?utm_source=github&utm_medium=readme&utm_content=docs_starter)

## 🏆 Why HugoBlox?

> *Why not just use WordPress, Webflow, or an AI builder like Lovable?*

| | **AI builders** (Lovable, v0) | **CMS platforms** (WordPress, Webflow) | **HugoBlox** |
| :--- | :---: | :---: | :---: |
| AI generates your pages | Yes | No | **Yes** |
| You own the output as readable files | No — React code | No — database | **Yes — Markdown** |
| Free to host forever | No | No | **Yes** |
| Human-editable without the tool | Barely | No | **Yes — it's Markdown** |
| Open source | No | No | **Yes — MIT licensed** |

> [!IMPORTANT]
> Your content is plain Markdown files. No lock-in, no database, no vendor dependency. If you ever want to leave, take your files and go.

## FAQ

<details>
<summary><b>Do I need to know Hugo?</b></summary>
No. Build with Hugo Chat or edit visually with Ownable CMS. Hugo is the build engine — you don't need to touch it.
</details>

<details>
<summary><b>Why use this instead of a purpose-built template?</b></summary>
If you know what you want — a CV, a blog, a docs site — use those templates. The starter is for when you want full control, or when you're building something that doesn't fit a predefined layout. It's also the ideal starting point for Hugo Chat, which generates exactly the pages you describe.
</details>

<details>
<summary><b>Can I add any HugoBlox block?</b></summary>
Yes. The starter includes the full HugoBlox block library — hero, features, stats, testimonials, pricing, CTA, collection, and more. Assemble them in any combination.
</details>

<details>
<summary><b>Can I host for free?</b></summary>
Yes. GitHub Pages, Netlify, Vercel, and Cloudflare Pages all have free tiers for static sites. This template ships with GitHub Pages deployment pre-configured.
</details>

<details>
<summary><b>Can I migrate my content later?</b></summary>
Yes. Your site is just Markdown files in a folder. Copy them anywhere.
</details>

<details>
<summary><b>What's Hugo Chat?</b></summary>
An AI assistant that understands Hugo and HugoBlox. Describe what you want and it generates the right pages with correct front matter, blocks, and content. <a href="https://hugo.chat/?utm_source=github&utm_medium=readme&utm_content=faq_starter">Free to try.</a>
</details>
